#### add `mc diagnostics` command for changeset context inspection

Add a first-class `mc diagnostics` command and `DiagnoseChangesets` CLI step type.

The command surfaces per-changeset context for every pending `.changeset/*.md` file, including the introduced and last-updated commit SHAs, linked review requests, and related issues. Inputs accept explicit changeset paths (absolute, `.changeset/`-relative, or short name), with deduplication and fallback resolution built in.

Output supports `--format text` and `--format json`. The JSON envelope includes `requestedChangesets` and a full `changesets` array with `path`, `summary`, `details`, `targets`, and `context` fields, making it suitable for scripting, CI, and AI agent consumption.

The command is included in default configs generated by `mc init` and documented in `docs/src/guide/11-diagnostics.md`.
