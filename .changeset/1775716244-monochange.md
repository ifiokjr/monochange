# split monochange internals into focused modules

`monochange` now keeps changeset helpers, changelog rendering, versioned-file updates, release artifact generation, and CLI runtime execution in dedicated modules. The `mc` behavior stays the same, but the top-level crate entry point is now much smaller and easier to navigate.

Before:

```rust
monochange::add_change_file(root, package_refs, bump, version, reason, change_type, details, output)
```

After:

```rust
monochange::add_change_file(
    root,
    monochange::AddChangeFileRequest::builder()
        .package_refs(package_refs)
        .bump(bump)
        .reason(reason)
        .version(version)
        .change_type(change_type)
        .details(details)
        .output(output)
        .build(),
)
```
